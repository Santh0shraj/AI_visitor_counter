# Face Tracker

## 1. Project Overview
This project is an automated face tracking and recognition system designed to monitor and catalog unique visitors from a video stream. It leverages modern deep learning models for detection and feature extraction to maintain accurate session and historical counts of individuals. A lightweight SQLite database ensures data persistence while an optional Flask dashboard provides real-time traffic statistics.

## 2. Architecture
- **`detector.py`**: Utilizes YOLOv8 (yolov8n-face.pt) to locate and extract precise bounding boxes for faces within a video frame.
- **`recognizer.py`**: Uses InsightFace (buffalo_l model) to generate 512-dimensional structural embeddings from the cropped face regions and computes cosine similarities.
- **`tracker.py`**: A custom, lightweight state engine that maintains active records of faces across consecutive frames using a green/yellow box tracking system to determine when visitors enter or leave the view.
- **`database.py`**: Manages all SQLite interactions using WAL mode, storing encoded facial embeddings in a `face_embeddings` table for multi-embedding pose stability and logging entry/exit events safely.
- **`logger.py`**: Handles generating formatted text logs for the console and log files, as well as cropping and saving physical snapshot images of faces upon entry and exit.
- **`counter.py`**: Tracks in-memory session counts of recognized visitors while interfacing with the database layer to retrieve absolute global unique visitor metrics.
- **`main.py`**: The central controller loop that integrates the components, manages configuration loading, handles video framing, coordinates inference, and orchestrates the tracking logic. Supports RTSP live streams with buffer flushing and output video generation with bounding boxes.

## 3. Setup Instructions
- `pip install -r requirements.txt`
- Place your video file as `sample.mp4` in the project root
- `python main.py` to run
- `python app.py` to run the optional dashboard

## 4. config.json explained

| Key | Type | Description |
|---|---|---|
| `video_source` | String | Path to the input video file (e.g., `"sample.mp4"`). |
| `use_rtsp` | Boolean | Boolean to switch between file and RTSP mode. |
| `rtsp_url` | String | Full RTSP camera URL for live stream. |
| `rtsp_reconnect_attempts` | Integer | Retry count on stream drop. |
| `frame_skip_interval` | Integer | Specifies how many frames to skip between expensive detection inferences. |
| `similarity_threshold` | Float | The minimum cosine similarity score required to classify two embeddings as the same person. |
| `exit_timeout_frames` | Integer | The number of consecutive absent frames before an active track is considered "exited". |
| `log_dir` | String | Root directory for storing text logs and image crop folders. |
| `db_path` | String | The persistent file path pointing to the SQLite tracking database. |
| `detection_confidence` | Float | The minimum confidence score generated by YOLO to accept a bounding box as a face. |
| `imgsz` | Integer | YOLO inference image size (standard is 640). |
| `output_video` | String | Output file path for annotated video. |
| `show_display` | Boolean | Show live window during processing. |

## 6. Assumptions made
- YOLOv8n-face model is used for detection
- InsightFace `buffalo_l` model is used for recognition
- Similarity threshold is 0.35 to distinguish faces
- Multi-embedding pose stability implemented (multiple embeddings per ID)
- Output video saved as `output.mp4`
- RTSP live stream supported with auto-reconnect logic
- Green boxes show active detection (directly from YOLO)
- Yellow boxes show passive tracking between detections
- SQLite WAL mode ensures crash-safe writes
- A face is considered exited if unseen for `exit_timeout_frames` consecutive frames

## 7. Compute load estimates

| Module | CPU Usage | GPU Usage | Notes |
|---|---|---|---|
| `FaceDetector` (YOLO) | Medium | High | Performs dense spatial convolutions on full frames. Skip intervals heavily mitigate load. |
| `FaceRecognizer` (InsightFace) | High | Low (CPU Provider) | Set to `CPUExecutionProvider` specifically to decouple from GPU bounds. |
| `FaceTracker` | Low | None | Simple mathematical dictionary lookups and bounds checks. |
| `DatabaseManager` | Low | None | SQLite writes efficiently on Disk IO, especially configured via WAL mode transactions. |
| Images / Logger | Low | None | OpenCV array cropping and JPEG encodes minimally utilize CPU. |

## 8. AI Planning and Architecture
The system was designed with strict modularity in mind, separating deep learning inferences from business logic and data storage. By isolating the complex tracking elements into simple, testable classes, the `main.py` loop remains procedural and straightforward.
